# Audio and Video Deepfake Detection

Audio and Video Deepfake Detection is a web application that uses machine learning and deep learning techniques to analyze audio and video files and determine whether they are real or generated by deepfake algorithms. The system helps users detect manipulated content and safeguard against synthetic media.

# Features

- User registration and login system.
- Audio and Video file upload for analysis.
- Audio analysis using audio features like MFCC, mel spectrogram, chroma features, zero crossing rate, spectral centroid, and spectral flatness.
- Video analysis by frame extraction and deepfake detection using trained VGG and ResNet models.
- Classification of audio as either real or deepfake.
- Real-time feedback with optional user contribution to improve model accuracy.
- Visual representation of the analysis results.
- Hash-based duplicate upload detection (same file returns saved prediction instantly).
- In-memory cache for faster repeat responses.
- Persistent prediction records (MongoDB Atlas via `MONGO_URI` or local SQLite fallback).
- Feedback pipeline that appends extracted audio features and corrected label to `dataset.csv` and `dataset_old.csv`.

# Setup and Deployment

## Local Development
1. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

2. Configure MongoDB (optional):
   - Copy `.env.example` to `.env` and add your MongoDB Atlas connection string:
     ```bash
     cp .env.example .env
     # Edit .env and set MONGO_URI with your Atlas credentials
     ```
   - If `MONGO_URI` is omitted, the app will use local SQLite instead.

3. Run the app:
   ```bash
   python app.py
   ```
   The app will load environment variables from `.env` automatically.

## Production Deployment
- Set the `MONGO_URI` environment variable on your production host (e.g., Heroku config vars, Docker secrets).
- The app will auto-detect and use Mongo if available; otherwise falls back to SQLite.

# Feedback Retraining Flow

- After a prediction, users can submit feedback from the result page.
- On feedback submission, the system:
	- fetches saved feature vectors for that upload,
	- determines corrected label from Yes/No feedback,
	- appends the row to the training CSV files,
	- updates the stored prediction so future duplicate uploads return the latest effective label quickly.

# Usage
1.Register a new account or log in with existing credentials.
2.Upload an audio or video file for analysis.
3.Wait for the analysis to complete.
4.View the result, indicating whether the uploaded file is classified as real or a deepfake.
5.Repeat the process for additional audio or video files.
